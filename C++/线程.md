使用线程需要包含头文件 `thread`

>[!CAUTION] 注意  
>启用的线程，一定要使用 `join` 函数等待结束  
>`join` 函数会将线程销毁

```cpp
#include <iostream>
#include <thread>
// 信号，为了停止 doWork 这个线程
static bool s_doWorkFinish = false;

void doWork()
{
	std::cout << "started thread id = " << std::this_thread::get_id() << std::endl;
	// 为了能使用 1s 这个字面量
	using namespace std::literals::chrono_literals;
	
	while (!s_doWorkFinish)
	{
		std::cout << "Working ..." << std::endl;
		std::this_thread::sleep_for(1s);
	}
}

int main()
{
	std::cout << "started thread id = " << std::this_thread::get_id() << std::endl;
	std::thread worker(doWork);
	
	std::cin.get();
	s_doWorkFinish = true;
	// 等待这个线程结束，不结束会阻塞线程
	worker.join();
	
	std::cout << "Finished." << std::endl;
	return 0;
}
```

# 使用线程加快运行速度

需要多线程完成的函数参数列表中的数据，一定要在这个函数结束之前都是能访问的，不要出现这个函数没结束，但是传进来的参数已经没了

对在异步中的写入操作，需要上锁

```cpp
#include <iostream>
#include <fstream>
#include <string>
#include <vector>
#include <future>

class Person
{
public:
	static Person Load(const std::string&filepath);
};

static std::vector<Person> m_Person;
// void 是函数返回类型
static std::vector<std::future<void>> m_Futues;
// 锁
static std::mutex s_PersonMutex;

// 一些不影响主程序的加载等部分
void func(const std::string& filepath)
{
	auto person = Person::Load(filepath);
	// 上锁
	// 作用域结束会自动解锁
	std::lock_guard<std::mutex> lock(s_PersonMutex);
	m_Person.push_back(person);
}

int main()
{
	std::ifstream stream("res/data/txt");
	std::string line;
	std::vector<std::string> dataFilepaths;
	// 将所有文件路径加载进来
	while(std::getline(stream,line))
		dataFilepaths.push_back(line);
	// 将所有文件路径下的资源加载进来
	for(const auto& filepath : dataFilepaths)
		m_Futues.push_back(std::async(std::launch::async,func,filepath));
}
```
