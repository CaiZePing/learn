主要用来解决命名冲突问题  
在 C 中只能使用 独特的 函数命名，才容易规避函数命名冲突，但是有时还是可能会发生  
但是在 C++ 中良好的使用 命名空间可以规避函数命名冲突  
但是如果滥用，就会 [[using namespace|这样]]

```cpp
#include <iostream>
#include <string>

namespace apple {
	void print(const std::string & text){
		std::cout << text << std::endl;
	}
}

namespace orange { namespace functions {
	// 这是一个恶意程序
	void print(const char* text){
		std::string temp = text;
		std::reverse(temp.begin(),temp.end());
		std::cout << temp << std::endl;
	}
}}

int main()
{
	{
		// 在这个作用域中可以使用 a 来代替 apple
		namespace a = apple;
		a::print("hello");
	}
	{
		// 使用 orange 中的 functions 命名空间
		orange::function::print("hello");
		// 也可以这样使用
		namespace o = orange::functions;
		o::print("hello");
		// 还可以这样使用
		using namespace orange::functions;
		print("hello");
	}
	
	return 0;
}
```
